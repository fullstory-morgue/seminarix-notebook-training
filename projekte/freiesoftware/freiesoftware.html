Projekt

Software muss frei sein - muss sie?

Problem/Frage

Freie Software ist vielen Nutzern weithin unbekannt, weil die Versorgung von Rechnern, die bereits komplett mit Software versorgt sind, sehr hoch ist. Dann: Never change a running system: Läuft das System mit nur wenig Abstürzen, ist das Motiv, sich mit dem Thema auseinander zu setzen, eher gering. Geht es allerdings um die Beschaffung und legale Nutzung weiterer Anwendungen, wird die Thematik interessanter.

Ziele
(Schwierigkeitsstufen)

Bedingungen freier (im Sinne von quelloffener)  Software kennen, den Unterschied zu proprietärer Software, zu lediglich kostenfreier Software kennen.
Argumentationsfiguren für den Einsatz quelloffener Software im Bildungsbreich kritisch abwägen können. Abhängigkeitsstrukturen ökonomischer Art würdigen können.

Durchführung
(einschließlich verwendeter Materialien / Software / Hardware)

Erste Phase dient der Information: Was ist freie Software, unter welchen Bedingungen wird sie hergestllt, wie und von wem wird sie weiter entwickelt., kann sie qualitativ mit proprietärer grundsätzlich oder nur in Teilen mithalten?
Zweite Phase: An einem konkreten Produkt (Mozilla, OpenOffice, Debian) werden die Entwicklungs- und Beteiligungslinien erarbeitet, im Kontakt zur Community werden Ideen / Wünsche diskutiert.
Dritte Phase: In der konkreten Umwelt die Bedingungen von Einsatz und Kosten von Software in der Ausbildung (etwa für eine Kommune) recherchieren.

Alternativen

An der Diskussion einer Mailingliste teilnehmen.
Über einen IRC-Chat (sidux) Fragen stellen und (hoffentlich) beantwortet bekommen.
